# 🌞 نظام التحكم الأوتوماتيكي بأحمال أنظمة الطاقة الشمسية

## الوصف
دارة تحكم ذكية مصممة للتحكم الأوتوماتيكي بالأحمال المتصلة بأنظمة الطاقة الشمسية. يقوم النظام بمراقبة جهد البطاريات والشبكة العامة واتخاذ قرارات ذكية لتشغيل أو إيقاف الأحمال تلقائياً.

## الميزات الرئيسية

### 🔋 التحكم الذكي بالأحمال
- **مراقبة جهد البطارية**: قراءة دقيقة لجهد البطارية مع نظام معايرة متقدم
- **حساس الشبكة العامة**: مراقبة توفر واستقرار الكهرباء من الشبكة العامة
- **تحكم في مخرجين ريلاي**: إدارة مستقلة لحملين مختلفين
- **منع تكرار الإقلاع**: نظام ذكي لمنع التشغيل والإيقاف المتكرر

### ⏰ التايمرات اليومية
- **3 تايمرات لكل مخرج**: تحديد فترات تشغيل متعددة في اليوم
- **جدولة مرنة**: تحديد أوقات البداية والنهاية بدقة
- **إيقاف آمن**: إيقاف التايمرات عند إعادة التشغيل لضمان الأمان

### 🌐 واجهة ويب عربية
- **تصميم عصري ومتجاوب**: يعمل على الهواتف وأجهزة الكمبيوتر
- **مراقبة فورية**: عرض البيانات الحالية للنظام
- **إعدادات شاملة**: تحكم كامل في جميع معايير النظام
- **معايرة تفاعلية**: إضافة وتعديل نقاط المعايرة

### 💾 حفظ الإعدادات
- **تخزين في EEPROM**: حفظ جميع الإعدادات في الذاكرة غير المتطايرة
- **استرجاع تلقائي**: تحميل الإعدادات عند إعادة التشغيل
- **نسخ احتياطي**: إمكانية إعادة ضبط المصنع

## المواصفات التقنية

### الأجهزة المطلوبة
- **ESP8266**: المتحكم الرئيسي
- **ريلايات 30 أمبير**: مخرجين للتحكم بالأحمال
- **حساس الشبكة العامة**: مراقبة توفر الكهرباء
- **مقسم جهد**: لقراءة جهد البطارية

### المداخل والمخارج
```
GPIO 14 → الريلاي الأول
GPIO 5  → الريلاي الثاني
GPIO 4  → حساس الشبكة العامة
A0      → قراءة جهد البطارية
```

### المتطلبات البرمجية
```cpp
// المكتبات المطلوبة
#include <ESP8266WiFi.h>      // إدارة WiFi
#include <ESP8266WebServer.h> // خادم الويب
#include <EEPROM.h>           // تخزين الإعدادات
#include <ArduinoJson.h>      // معالجة JSON
```

## التركيب والإعداد

### 1. تحضير البيئة البرمجية
1. تثبيت Arduino IDE
2. إضافة لوحة ESP8266:
   ```
   File → Preferences → Additional Board Manager URLs:
   http://arduino.esp8266.com/stable/package_esp8266com_index.json
   ```
3. تثبيت مكتبة ArduinoJson:
   ```
   Tools → Manage Libraries → Search: ArduinoJson
   ```

### 2. تحميل الكود
1. فتح ملف `Solar_System_Controller.ino`
2. اختيار اللوحة: `NodeMCU 1.0 (ESP-12E Module)`
3. تحديد المنفذ الصحيح
4. رفع الكود

### 3. التوصيلات الكهربائية
```
ESP8266          ←→    الجهاز
========================================
GPIO 14         ←→    ريلاي 1 (Signal)
GPIO 5          ←→    ريلاي 2 (Signal)
GPIO 4          ←→    حساس الشبكة العامة
A0              ←→    مقسم جهد البطارية
VCC             ←→    +5V
GND             ←→    Ground
```

## نظام المعايرة

### أهمية المعايرة
يستخدم النظام جدول معايرة متقدم لتحويل قراءات ADC إلى قيم جهد دقيقة، مما يضمن:
- دقة قياس 0.1 فولت
- تعويض عدم الخطية في مقسم الجهد
- تحسين الأداء عبر نطاق واسع من الجهود

### جدول المعايرة الافتراضي
```cpp
قراءة ADC    ←→    الجهد الحقيقي
================================
0.00         ←→    0.000 V
78.0         ←→    5.016 V
102.0        ←→    6.530 V
133.0        ←→    8.400 V
147.0        ←→    9.120 V
161.0        ←→    10.040 V
198.0        ←→    12.430 V
240.0        ←→    15.070 V
317.0        ←→    19.980 V
368.0        ←→    23.270 V
399.0        ←→    25.160 V
522.0        ←→    32.770 V
```

### إضافة نقاط معايرة جديدة
1. قياس الجهد الحقيقي بمقياس دقيق
2. تسجيل قراءة ADC من الواجهة
3. إدخال القيمتين في صفحة المعايرة
4. الضغط على "إضافة نقطة"

## نظام التايمرات

### كيفية العمل
- **3 تايمرات لكل ريلاي**: يمكن ضبط 3 فترات تشغيل مختلفة
- **تشغيل مستقل**: كل تايمر يعمل بشكل منفصل
- **أولوية عالية**: التايمرات تعمل حتى مع انخفاض الجهد أو انقطاع الشبكة
- **إيقاف آمن**: عند إعادة التشغيل، تتوقف جميع التايمرات

### مثال على الاستخدام
```
الريلاي 1:
- التايمر 1: 06:00 → 08:00 (تشغيل الإضاءة صباحاً)
- التايمر 2: 12:00 → 14:00 (تشغيل المراوح ظهراً)
- التايمر 3: 18:00 → 22:00 (تشغيل الإضاءة مساءً)

الريلاي 2:
- التايمر 1: 07:00 → 17:00 (تشغيل المبرد نهاراً)
- التايمر 2: معطل
- التايمر 3: معطل
```

## المنطق التشغيلي

### شروط تشغيل الريلايات
1. **النظام مفعل** (يمكن تعطيله من الإعدادات)
2. **الشبكة العامة متوفرة ومستقرة** (لمدة دقيقة متواصلة)
3. **أحد الشروط التالية**:
   - جهد البطارية أعلى من جهد التشغيل المحدد
   - التايمر مفعل ونحن في الفترة الزمنية المحددة
   - التحكم اليدوي مفعل

### نظام منع التكرار
- مراقبة استقرار الشبكة العامة لمدة دقيقة كاملة
- تأخير بين تشغيل الريلايات (افتراضي: 3 ثواني)
- منع التشغيل والإيقاف المتكرر

## استكشاف الأخطاء

### مشاكل شائعة وحلولها

#### لا يمكن العثور على الشبكة
- **التحقق**: هل تم رفع الكود بنجاح؟
- **الحل**: إعادة تشغيل ESP8266

#### قراءات جهد غير دقيقة
- **التحقق**: هل تم معايرة النظام؟
- **الحل**: إضافة نقاط معايرة جديدة

#### الريلايات لا تعمل
- **التحقق**: 
  - هل النظام مفعل؟
  - هل الشبكة العامة مستقرة؟
  - هل الجهد أعلى من جهد التشغيل؟
- **الحل**: فحص الإعدادات والتوصيلات

#### الواجهة لا تستجيب
- **الحل**: إعادة تشغيل النظام من صفحة النظام

### رسائل السجل
يمكن مراقبة النظام عبر Serial Monitor (115200 baud):

النظام جاهز للعمل ✅

استقرار الشبكة: مستقرة

### حدود النظام
- **تيار الريلاي**: 30 أمبير كحد أقصى لكل ريلاي
- **جهد التشغيل**: 12V, 24V, 48V DC
- **درجة الحرارة**: -20°C إلى +70°C
- **الرطوبة**: أقل من 85%
